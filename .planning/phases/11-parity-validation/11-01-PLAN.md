<task type="auto">
  <name>Task 2: Create label filtering equivalence validation script</name>
  <files>scripts/parity-labels.sh</files>
  <action>
Create `scripts/parity-labels.sh` that validates label filtering produces equivalent test sets between Ginkgo and Gauge:

1. **For each label category** (sanity, smoke, e2e, disconnected):
   - Run `ginkgo run --dry-run -v --label-filter="$label" ./tests/... 2>&1` and extract test names
   - Extract Polarion test case IDs (PIPELINES-XX-TCXX pattern) from the output
   - Extract equivalent test set from Gauge using `gauge list scenarios --tags "$label" specs/` or by parsing spec files for tag annotations
   - Compare the Polarion test case ID sets (sorted)
   - Report: matching count, missing in Ginkgo, extra in Ginkgo

2. **For compound expressions**:
   - Test `e2e && !disconnected` (Ginkgo) vs `e2e & !disconnected` (Gauge)
   - Test `sanity || smoke` (Ginkgo) vs `sanity | smoke` (Gauge)
   - Same comparison logic as above

3. **Output format**:
   ```
   Label Filtering Parity Report
   ==============================
   Label: sanity
     Ginkgo count: N
     Gauge count:  M
     Status: MATCH / MISMATCH
     [If mismatch: list of differing test IDs]

   Label: smoke
     ...

   Compound: e2e && !disconnected
     ...

   Overall: PASS / FAIL
   ```

4. Make the script executable. Exit 0 on all labels matching, exit 1 on any mismatch.

Key considerations:
- Gauge tags are specified in spec files with `tags: tagname` syntax. Parse these if Gauge CLI is not available.
- Ginkgo label filtering is hierarchical -- a spec inherits labels from its parent Describe/Context. The validation must account for this.
- Normalize test identifiers to Polarion IDs for comparison since test name formats differ between frameworks.
  </action>

    <automated>test -x scripts/parity-labels.sh && grep -q "label-filter" scripts/parity-labels.sh && grep -q "sanity" scripts/parity-labels.sh && grep -q "disconnected" scripts/parity-labels.sh</automated>
  </verify>
  <done>parity-labels.sh validates all four label categories plus compound expressions. Reports per-label match/mismatch with specific test IDs for any divergences. Exits 0 on full match.</done>
</task>
