**Step 2: Create tests/operator/autoprune_test.go (15 tests)**

Create a top-level `Describe("PIPELINES-12: Verify auto-prune E2E", Serial, Label("e2e", "operator", "auto-prune", "admin"), func() {...})`.

Inside, add a `BeforeEach` that calls `operator.ValidateOperatorInstallStatus(sharedClients, store.GetCRNames())`.

Create the following local helper functions at the top of the file (unexported, file-scoped):

1. `removePrunerConfig()` -- Patches TektonConfig to remove pruner config:
   ```go
   cmd.MustSucceed("oc", "patch", "TektonConfig", "config", "--type=json",
       "-p", `[{"op":"remove","path":"/spec/pruner"}]`)
   ```
   Wrap in a recovery since pruner may not exist. Wait for TektonConfig to be ready after.

2. `updatePrunerConfig(keep, schedule, resources, keepSince string, withKeep, withKeepSince bool)` -- Patches TektonConfig with pruner configuration. Build the JSON patch dynamically based on which fields are "with" vs "without". Wait for TektonConfig to become ready after patch.

3. `updatePrunerConfigWithInvalidData(keep, schedule, resources, keepSince string, withKeep, withKeepSince bool) string` -- Same as above but uses `cmd.Run()` (not MustSucceed) and returns the error output.

4. `assertCronjobPresence(prefix, targetNamespace string, shouldBePresent bool)` -- Uses Eventually to poll for cronjob with given prefix in the namespace. Timeout: 2 minutes.

5. `assertResourceCount(resourceType string, expectedCount int, timeoutSeconds int)` -- Uses Eventually to poll `oc get <resourceType> -n <namespace>` and count results. Matches the Gauge step "N pipelinerun(s)/taskrun(s) should be present within M seconds".

6. `storeCronjobName(namespace, schedule string) string` -- Gets the name of the cronjob with the given schedule in the namespace.

Create 15 `It` blocks inside individual `Context` blocks (each Context is `Ordered` for multi-step tests). Each test:
- Starts by calling `removePrunerConfig()` in the first step
- Creates resources from testdata fixtures using `oc.Create()`
- Configures pruner and waits for results
- Uses DeferCleanup to remove pruner config at end

Detailed test mapping:

**TC01** (`Context("Verify auto prune for taskrun", Ordered, func() {...})`):
- Remove pruner config, create 4 resources, set pruner keep=2 for taskrun, assert cronjob present
- Eventually: 2 taskruns within 180s, 5 pipelineruns within 120s
- DeferCleanup: remove pruner config, assert cronjob not present

**TC02-TC03**: Same pattern, different resources (pipelinerun, both)

**TC04**: Keep-since strategy -- create resources, sleep replaced with `time.Sleep(120*time.Second)` (EXCEPTION: this sleep is intentional -- it creates time gap between old and new resources for keep-since pruning, NOT waiting for a condition). Create more resources, set pruner with keep-since=2, verify counts.

**TC05**: Skip namespace annotation -- annotate namespace with prune.skip=true, verify resources NOT pruned, remove annotation, verify pruning resumes.

**TC06-TC07**: Per-namespace resource annotation overrides.

**TC08-TC09**: Per-namespace keep/keep-since with different global strategies.

**TC10**: Per-namespace keep override with global keep-since strategy, then add strategy annotation.

**TC11**: Per-namespace schedule annotation override.

**TC12**: Validation tests -- 3 invalid config attempts, each expecting specific error message. Use `cmd.Run()` and assert stderr contains expected error.

**TC13**: Cronjob stability -- set pruner, store cronjob name, add random annotation, verify name unchanged, remove annotation, verify unchanged, add label, verify unchanged, remove label, verify unchanged.

**TC14**: Container count -- set pruner, create 2 projects, assert cronjob containers=1, DeferCleanup deletes projects and removes pruner.

**TC15**: Namespace deletion stability -- create namespace with pruner annotation, verify cronjob, delete namespace, verify cronjob gone, verify operator still installed, repeat with second namespace.

IMPORTANT: Replace ALL `Sleep for "N" seconds` with appropriate patterns:
- For the timing gap in TC04/TC09 (intentional delay to create old resources): Keep as `time.Sleep()` but add a comment explaining WHY this is intentional (not polling-replaceable)
- For TC11 `Sleep for "60" seconds`: Replace with Eventually polling for the expected resource count
- For TC13 `Sleep for "5" seconds`: Replace with `time.Sleep(5*time.Second)` (brief wait for annotation propagation; not a polling scenario) with comment
- For TC14 `Sleep for "10" seconds`: Replace with Eventually polling for cronjob to appear
- For TC15 `Sleep for "5" seconds`: Replace with Eventually polling for cronjob to disappear

Label sanity-tagged tests (TC01, TC04, TC07, TC14, TC15) with `Label("sanity")`.
