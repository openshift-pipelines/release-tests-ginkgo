   - `GetPipelineName` / `GetPipelineNameFromMR` should accept `(c *clients.Clients, namespace string, projectID, mrID int)` and return `(string, error)`
   - `TriggerPushOnForkMain` should accept `(projectID int)` and return `error`
   - `AddComment` should accept `(projectID, mrID int, comment string)` and return `error`
   - `AddLabel` should accept `(projectID, mrID int, label, color, description string)` and return `error`
   - `CleanupPAC` should accept `(c *clients.Clients, namespace string, projectID int, smeeDeploymentName string)` and return `error`
   - `UpdateAnnotation` should accept the key/value and return the updated content or error

3. **Imports:** Use ONLY `github.com/openshift-pipelines/release-tests-ginkgo/pkg/*` for local packages. Keep `github.com/xanzy/go-gitlab` for GitLab client (not migrating now per roadmap decision DEP-02). Keep PAC API imports (`pipelines-as-code`) as-is.

5. **Constants:** Port all constants from reference (initialBackoffDuration, maxRetries, targetURL, webhookConfigName, file paths).

6. **Do NOT import `pkg/store`** from the new functions. The test spec layer will manage state via closure variables in Ordered containers.
