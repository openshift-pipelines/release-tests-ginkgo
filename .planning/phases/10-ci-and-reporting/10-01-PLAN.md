<task type="auto">
  <name>Task 2: Create test runner wrapper script</name>
  <files>scripts/run-tests.sh</files>
  <action>
Create `scripts/run-tests.sh` as an executable bash script that standardizes Ginkgo test invocation. The script must:

1. **Accept a mode argument** as the first positional parameter:
   - `sanity` -- runs `--label-filter="sanity"`
   - `smoke` -- runs `--label-filter="smoke"`
   - `e2e` -- runs `--label-filter="e2e"`
   - `e2e-connected` -- runs `--label-filter="e2e && !disconnected"`
   - `disconnected` -- runs `--label-filter="disconnected"`
   - `all` -- no label filter (runs everything)
   - Custom string -- passed directly as `--label-filter` value
   - No argument -- defaults to `all`

2. **Always set these flags**:
   - `--junit-report=junit-report.xml` -- JUnit output
   - `--output-dir=${ARTIFACTS_DIR:-./artifacts}` -- output directory (CI systems set ARTIFACTS_DIR)
   - `--timeout=${GINKGO_TIMEOUT:-4h}` -- explicit timeout, overridable via env var
   - `--randomize-all` -- randomize spec order to surface hidden dependencies
   - `-v` -- verbose output for CI log readability

3. **CI mode** (when `CI=true` env var is set or `--ci` flag is passed):
   - Add `--fail-on-focused` -- prevent FDescribe/FIt from passing CI
   - Add `--no-color` -- clean CI logs

4. **Parallel mode** (when `--parallel` or `-p` flag is passed):
   - Add `--procs=${GINKGO_PROCS:-4}` -- configurable parallelism
   - Default to 4 processes, overridable via GINKGO_PROCS env var

5. **Test path**: Default to `./tests/...` but allow override via `--path=<path>` flag or remaining positional args after mode.

6. **Usage/help**: Display usage when `--help` or `-h` is passed.

7. **Create artifacts directory** if it does not exist.

8. **Print the full ginkgo command** before executing it (for CI log transparency).

Script structure:
```bash
#!/usr/bin/env bash
set -euo pipefail

# Parse arguments
# Build ginkgo command
# Print and execute
```

Make the script executable: `chmod +x scripts/run-tests.sh`.

Example invocations the script should support:
- `./scripts/run-tests.sh sanity` -- run sanity tests
- `./scripts/run-tests.sh e2e --ci --parallel` -- CI parallel e2e
- `./scripts/run-tests.sh "operator && sanity" --ci` -- custom label in CI
- `GINKGO_TIMEOUT=2h ./scripts/run-tests.sh all` -- custom timeout
  </action>
