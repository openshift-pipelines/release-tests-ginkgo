**DescribeTable 1: "Ecosystem Task Pipelines" (simple create-verify pattern)**

Table body function signature: `func(testcaseID, pipelineRunName string, resources []string, expectedStatus string)`

Body logic:
1. Create unique namespace: `ns := fmt.Sprintf("eco-%s-%s", sanitizedTestName, uuid[:8])`
2. `oc.CreateNewProject(ns)` then `DeferCleanup(oc.DeleteProjectIgnoreErors, ns)`
3. Create shared clients for namespace: `sharedClients.NewClientSet(ns)`
4. Wait for service account "pipeline" to exist in namespace (use k8s.WaitForServiceAccount if available, otherwise a simple poll)
5. Loop over `resources` slice: `oc.Create(resource, ns)` for each
6. `pipelines.ValidatePipelineRun(sharedClients, pipelineRunName, expectedStatus, ns)`

Labels: `Label("ecosystem", "e2e")`

**DescribeTable 3: "Multiarch Ecosystem Task Pipelines"**

Same structure as DescribeTable 1 but with architecture-specific Skip logic inside the body function.

Table body function: `func(testcaseID, pipelineRunName string, resources []string, expectedStatus string, requiredArchs []string)`

Body adds Skip logic:
```go
if len(requiredArchs) > 0 {
    archMatch := false
    for _, arch := range requiredArchs {
        if config.Flags.ClusterArch == arch {
            archMatch = true
            break
        }
    }
    if !archMatch {
        Skip(fmt.Sprintf("requires one of architectures: %v (cluster is %s)", requiredArchs, config.Flags.ClusterArch))
    }
}
```

Note: `requiredArchs` is a `[]string` literal, which is safe for tree-construction-time evaluation.
