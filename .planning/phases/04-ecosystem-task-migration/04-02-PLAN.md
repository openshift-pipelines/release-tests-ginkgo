**Part B: Cache Pipeline Tests (2 tests) -- Ordered container**

These tests use `opc.StartPipeline` to start pipelines with specific params and validate log content. They run the pipeline twice to verify cache behavior (first upload, then no-op). Use `Ordered` container because the second run depends on the first having populated the cache.

```go
Describe("Ecosystem Cache Tasks", Label("ecosystem", "e2e", "cache"), func() {

    It("cache-upload stepaction: PIPELINES-29-TC15", Label("sanity"), func() {
        ns := createTestNamespace("eco-cache-upload")
        DeferCleanup(oc.DeleteProjectIgnoreErors, ns)
        sharedClients.NewClientSet(ns)

        // Create pipeline and PVC
        oc.Create("testdata/ecosystem/pipelines/cache-stepactions-python.yaml", ns)
        oc.Create("testdata/pvc/pvc.yaml", ns)

        // First run -- should upload cache
        params := map[string]string{"revision": "release-v1.17"}
        workspaces := map[string]string{"name": "source", "claimName": "shared-pvc"}
        // Note: workspace format may need adjustment to match opc.StartPipeline signature
        // The Gauge code uses "name=source,claimName=shared-pvc" format
        prName1 := opc.StartPipeline("caches-python-pipeline", params, workspaces, ns, "--use-param-defaults")
        pipelines.ValidatePipelineRun(sharedClients, prName1, "successful", ns)

        // Validate first run logs contain upload message
        logs1 := cmd.MustSucceed("oc", "logs", "-l",
            "tekton.dev/pipelineRun="+prName1+",tekton.dev/pipelineTask=cache-upload",
            "-n", ns).Stdout()
        Expect(strings.ToLower(logs1)).To(ContainSubstring("upload /workspace/source/cache/lib content to oci image"))

        // Second run -- should skip upload (cache already exists)
        prName2 := opc.StartPipeline("caches-python-pipeline", params, workspaces, ns, "--use-param-defaults")
        pipelines.ValidatePipelineRun(sharedClients, prName2, "successful", ns)

        logs2 := cmd.MustSucceed("oc", "logs", "-l",
            "tekton.dev/pipelineRun="+prName2+",tekton.dev/pipelineTask=cache-upload",
            "-n", ns).Stdout()
        Expect(strings.ToLower(logs2)).To(ContainSubstring("no need to upload cache"))
    })

    It("cache upload with revision change: PIPELINES-29-TC16", func() {
        ns := createTestNamespace("eco-cache-rev")
        DeferCleanup(oc.DeleteProjectIgnoreErors, ns)
        sharedClients.NewClientSet(ns)

        oc.Create("testdata/ecosystem/pipelines/cache-stepactions-python.yaml", ns)
        oc.Create("testdata/pvc/pvc.yaml", ns)

        workspaces := map[string]string{"name": "source", "claimName": "shared-pvc"}

        // First run with revision release-v1.17
        params1 := map[string]string{"revision": "release-v1.17"}
        prName1 := opc.StartPipeline("caches-python-pipeline", params1, workspaces, ns, "--use-param-defaults")
        pipelines.ValidatePipelineRun(sharedClients, prName1, "successful", ns)

        logs1 := cmd.MustSucceed("oc", "logs", "-l",
            "tekton.dev/pipelineRun="+prName1+",tekton.dev/pipelineTask=cache-upload",
            "-n", ns).Stdout()
        Expect(strings.ToLower(logs1)).To(ContainSubstring("upload /workspace/source/cache/lib content to oci image"))

        // Second run with different revision (master) -- should upload again
        params2 := map[string]string{"revision": "master"}
        prName2 := opc.StartPipeline("caches-python-pipeline", params2, workspaces, ns, "--use-param-defaults")
        pipelines.ValidatePipelineRun(sharedClients, prName2, "successful", ns)

        logs2 := cmd.MustSucceed("oc", "logs", "-l",
            "tekton.dev/pipelineRun="+prName2+",tekton.dev/pipelineTask=cache-upload",
            "-n", ns).Stdout()
        Expect(strings.ToLower(logs2)).To(ContainSubstring("upload /workspace/source/cache/lib content to oci image"))
    })
})
```

NOTE on workspace parameter format: The Gauge code uses `"name=source,claimName=shared-pvc"` which is parsed by the step implementation into `map[string]string{"name": "source", "claimName": "shared-pvc"}`. Check the actual `opc.StartPipeline` function signature to determine the correct format. The workspace map keys may actually be the workspace name as key and PVC claim as value (e.g., `map[string]string{"source": "claimName=shared-pvc"}`). The Gauge step code in `steps/cli/opc.go` splits on comma and uses the first part as key and second as value: `workspaces[strings.Split(workspaceValue, ",")[0]] = strings.Split(workspaceValue, ",")[1]`. So the format is `workspaces["name=source"] = "claimName=shared-pvc"`. Check the actual opc.StartPipeline to understand how it processes this map.

**Special handling for dotnet (TC02):** The dotnet test has a unique pattern where it computes an `EXAMPLE_REVISION` parameter based on the version number. In the Gauge code:
```
versionFloat >= 5.0 -> params["EXAMPLE_REVISION"] = "dotnet-" + versionStr
versionFloat < 5.0  -> params["EXAMPLE_REVISION"] = "dotnetcore-" + versionStr
```

Handle this by adding a conditional inside the DescribeTable body function that checks if `pipelineName` is `"s2i-dotnet-pipeline"` and adds the extra EXAMPLE_REVISION param. Alternatively, add an optional `extraParamFunc func(tag string) map[string]string` parameter to the Entry. The simplest approach: detect `imagestreamName == "dotnet"` inside the body and apply the dotnet-specific logic.

```go
// Inside table body, after setting params["VERSION"] = tag:
if imagestreamName == "dotnet" {
    re := regexp.MustCompile(`\d+\.\d+`)
    versionStr := re.FindString(tag)
    versionFloat, _ := strconv.ParseFloat(versionStr, 64)
    if versionFloat >= 5.0 {
        params["EXAMPLE_REVISION"] = "dotnet-" + versionStr
    } else {
        params["EXAMPLE_REVISION"] = "dotnetcore-" + versionStr
    }
}
```
