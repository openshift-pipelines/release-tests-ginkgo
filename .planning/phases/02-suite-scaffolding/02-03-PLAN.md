2. A spec demonstrating Skip based on cluster architecture:
```go
It("validates ARM-specific behavior", func() {
    if config.Flags.ClusterArch != "arm64" {
        Skip("requires ARM64 cluster (set ARCH=linux/arm64)")
    }
    // Actual test logic would go here
})
```

3. A spec demonstrating Skip based on missing environment variable:
```go
It("validates component version", Label("sanity"), func() {
    version := os.Getenv("PIPELINE_VERSION")
    if version == "" {
        Skip("PIPELINE_VERSION not set in environment")
    }
    // Actual version validation would go here
})
```

4. A `Context` block demonstrating Skip in BeforeEach (applies to all specs in the container):
```go
Context("when cluster is disconnected", func() {
    BeforeEach(func() {
        if !config.Flags.IsDisconnected {
            Skip("requires disconnected cluster")
        }
    })
    It("test A for disconnected", func() { /* ... */ })
    It("test B for disconnected", func() { /* ... */ })
})
```

5. Comments documenting:
   - Skip in It block: skips individual spec
   - Skip in BeforeEach: skips all specs in the Context/Describe container
   - Skipped specs appear as "S" in Ginkgo output and as "skipped" in JUnit XML
   - config.Flags is populated from environment variables and CLI flags (available after flag.Parse in testing.T)
